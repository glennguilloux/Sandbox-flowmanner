This is the structured architectural analysis of the Flowmanner backend codebase, based on a comprehensive review of the active services, configuration files, and implementation folders.

1. Architecture & Core Configuration
Core Framework & ASGI: The application is built on FastAPI, as configured in the main entrypoint main_fastapi.py:90-130.
ASGI Server & WebSocket Bypass: Real-time Socket.IO polling and WebSocket traffic are managed via ws_app main_fastapi.py:368-410. Because Starlette's BaseHTTPMiddleware disrupts WebSocket routing, a custom ASGI middleware, _WebSocketBypassMiddleware, is wrapped directly around the FastAPI instance in main_fastapi.py:370 to route /ws streams directly to python-socketio before normal HTTP middleware execution.
Environment Setup: Setting management is orchestrated by config.py via Pydantic model configurations.
CORS & Security Headers: Under production (settings.APP_ENV != "development"), CORS origins are strictly bound using settings.cors_origins_list with narrow allowances for methods and headers. A custom SecurityHeadersMiddleware is registered in main_fastapi.py:143 to inject necessary secure response headers.
Containerization: The production layout in docker-compose.yml defines a multi-container deployment. To prevent transient file overrides, the backend image is fully standard-baked via Dockerfile without loose host code-mounts, which requires complete image rebuilding and container recreation when shipping code updates.
2. API Layer
Router Organization: The api endpoints are partitioned in main_fastapi.py:338 by versioning schemes (api_v1_router and api_v2_router).
Dependency Injection: DB resource context is injected down requests using get_db from database.py:28-42. Hierarchical permissions and authentications are resolved inside Depends scopes via standard wrappers in deps.py:
get_current_user: Resolves bearer token credentials and yields active SQL database records of the user.
require_role: Checks role categories (e.g. admin, pro, free).
require_permission: High-granularity authorization resolver resolving system role hierarchies, custom roles, or active delegations.
Request/Response Models: Serialization and validation are managed by Pydantic models. Raw API outputs are enclosed in unified JSON wrappers via the custom _envelope helper.
OpenAPI & Theme Customization: In main_fastapi.py:210, OpenAPI schemas are decorated with dynamic list elements for header negotiation (e.g., Accept-Version parameter injection). Under development, Swagger /docs and Redoc /redoc pathways are styled dynamically with custom dark CSS templates (e.g., SWAGGER_DARK_CSS and REDOC_DARK_CSS).
3. Authentication & Authorization
OAuth & SSO Providers: Implemented using standard OIDC Authorization Code Flows with PKCE code challenges in oidc.py. Available federated sign-in methods are resolved via the OIDC provider configurations in oidc.py:50.
JWT & Session Flow: Access and refresh tokens are issued, tracked, and validated using family-bounded rotative models within auth.py:110. Token family reuse detection exists in refresh auth.py:340 to instantly revoke all subsidiary tokens of a user when an old refresh token is reused.
Security & Lockouts: Features 2FA (TOTP verification with backup code ingestion) and account locking via record_failed_login inside auth.py:182 to temporarily block bruteforce attacks on password credentials.
Homelab Tunnel Gateway: Traffic transitions from public interfaces (VPS at 74.208.115.142) down to the internal API (Homelab at 10.99.0.3:8000) over a secure WireGuard client interface (wg0).

Database Sessions:

Key Models:
