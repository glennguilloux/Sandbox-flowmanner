    not needed for the queries this code issues).

DISCREPANCY (not blocking, but worth noting):

  The prompt said "default limit 100, max 1000". The code uses
  DEFAULT_LIMIT = 1000, MAX_LIMIT = 1000. So the default equals the
  max. This is probably fine (callers can pass limit explicitly to
  get fewer), but it deviates from the prompt. Sub-agent chose
  default=1000 perhaps because the existing endpoint had no limit and
  returning the full stream by default is safer for the "show me
  everything" use case. Document the choice in a follow-up; do not
  block on it.
