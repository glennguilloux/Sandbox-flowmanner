## Key finding: the suppresses are NOT accumulation debt

Every single per-file comment explains a concrete, verifiable reason:
- **TCH001/002/003 (21 findings):** runtime-needed imports for FastAPI OpenAPI
  forward-ref resolution (`from __future__ import annotations` stringizes
  signatures; FastAPI calls `get_type_hints` at startup and would SILENTLY skip
  routes whose names are only in a `TYPE_CHECKING` block) and SQLAlchemy
  mapper-configuration-time resolution of `Mapped[datetime | None]`. Spot-checked
  `mission.py`: all flagged names appear ONLY in function-signature annotations
  (path params, `Depends`), exactly the documented pattern.
- **UP042 (3 findings):** intentional `str`+Enum for clean JSON serialization.
  **Verified at runtime** — `NodeType`, `InboxItemStatus`, `ClaimType` are all
  `issubclass(str)`. This is a deliberate API design, not debt.
- **F821 globs (entries 1-2):** currently mask ZERO findings (SQLAlchemy resolves
  forward refs lazily). The ignore is preventive, not masking active errors.
- **UP017 (entry 13):** pinned-env suppress, NOT stale. `datetime.UTC` exists at
  module level on 3.11, BUT the test does `from datetime import datetime` so
  `datetime` is the *class*, and `datetime.datetime.UTC` does not exist. A naive
  `datetime.now(datetime.UTC)` raises `AttributeError` under the canonical venv
  (verified: pytest FAILS). The commented "predates datetime.UTC" is slightly
  imprecise (UTC landed in 3.12, not absent from 3.11) but the practical
  conclusion — keep `timezone.utc` — is CORRECT. I attempted the fix, it broke
  the test, so it was reverted.
